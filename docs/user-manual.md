## 应用介绍

AI Manager 是一个轻量级、高性能的AI工具管理平台，专为简化AI服务管理而设计。从 Python/FastAPI 迁移到 Rust/Tauri 架构后，应用具备以下特点：

- **轻量级设计**: 安装包小于 15MB，启动时间小于 2 秒
- **高性能**: 内存占用小于 100MB，响应速度提升 90%
- **跨平台**: 支持 Windows、macOS、Linux 三个主要平台
- **数据安全**: 本地数据加密存储，保障用户隐私
- **中文优化**: 完整的中文界面和本地化支持

### 主要功能

- **Claude 供应商管理**: 管理 Anthropic Claude API 服务配置
- **Codex 供应商管理**: 管理各种代码生成服务配置
- **Agent 指导文件**: 管理AI助手的指导模板和提示词
- **MCP 服务器**: 管理模型控制协议服务器连接
- **通用配置**: 管理应用级别的配置项

## 系统要求

### 最低系统要求

**Windows**
- Windows 10 (1903) 或更高版本
- 2GB RAM (推荐 4GB 或更高)
- 100MB 可用磁盘空间

**macOS**
- macOS 10.13 (High Sierra) 或更高版本
- 2GB RAM (推荐 4GB 或更高)
- 100MB 可用磁盘空间

**Linux**
- 支持的发行版: Ubuntu 18.04+, CentOS 7+, Fedora 30+, Debian 10+
- 2GB RAM (推荐 4GB 或更高)
- 100MB 可用磁盘空间
- 以下依赖包: `libwebkit2gtk-4.0-37`, `libnotify4`, `libnss3`, `libxss1`, `libxtst6`

### 推荐系统配置

- **处理器**: 双核 2.0GHz 或更高
- **内存**: 4GB RAM 或更高
- **网络**: 稳定的互联网连接（用于AI服务API调用）
- **显示器**: 1024x768 分辨率或更高

## 安装指南

### Windows 安装

1. **下载安装包**
   - 从官方网站下载 `AI-Manager_0.1.0_x64-setup.exe`
   - 验证文件完整性（可选）

2. **运行安装程序**
   - 右键点击安装程序，选择"以管理员身份运行"
   - 按照安装向导提示完成安装
   - 选择安装目录（默认: `C:\Program Files\AI Manager`）

3. **首次启动**
   - 从开始菜单启动 AI Manager
   - 或者双击桌面快捷方式

### macOS 安装

1. **下载安装包**
   - 下载 `AI-Manager_0.1.0_x64.app.tar.gz` (Intel Mac)
   - 或 `AI-Manager_0.1.0_aarch64.app.tar.gz` (Apple Silicon Mac)

2. **解压并安装**
   ```bash
   tar -xzf AI-Manager_0.1.0_x64.app.tar.gz
   mv AI-Manager.app /Applications/
   ```

3. **首次启动**
   - 从 Launchpad 或 Applications 文件夹启动
   - 首次启动时可能需要在"系统偏好设置"中允许应用运行

### Linux 安装

1. **下载 AppImage (推荐)**
   ```bash
   wget https://github.com/ai-manager/migration/releases/download/v0.1.0/AI-Manager_0.1.0_amd64.AppImage
   chmod +x AI-Manager_0.1.0_amd64.AppImage
   ./AI-Manager_0.1.0_amd64.AppImage
   ```

2. **或者使用包管理器安装 (Debian/Ubuntu)**
   ```bash
   sudo dpkg -i ai-manager_0.1.0_amd64.deb
   sudo apt-get install -f  # 解决依赖问题
   ```

## 快速开始

### 1. 启动应用

首次启动 AI Manager 后，您将看到主界面：

```
┌─────────────────────────────────────────────────────────┐
│ AI Manager v0.1.0                                      │
│ ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐      │
│ │Claude   │ │Codex    │ │Agent    │ │MCP      │      │
│ │供应商   │ │供应商   │ │指导文件 │ │服务器   │      │
│ └─────────┘ └─────────┘ └─────────┘ └─────────┘      │
│                                                         │
│ ┌─────────────────────────────────────────────────────┐ │
│ │ 通用配置                                             │ │
│ └─────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────┘
```

### 2. 配置 Claude 供应商

1. 点击 "Claude 供应商" 标签页
2. 点击 "添加供应商" 按钮
3. 填写配置信息：
   - **名称**: 为供应商起一个易识别的名称
   - **URL**: `https://api.anthropic.com`
   - **Token**: 您的 Claude API 密钥
   - **模型**: 选择默认模型（如 claude-3-sonnet-20240229）
4. 点击 "保存" 按钮
5. 点击供应商卡片上的"启用"按钮激活配置

### 3. 测试连接

在保存配置后，可以点击"连接测试"按钮验证配置是否正确：

- ✅ **连接成功**: 配置正确，可以开始使用
- ❌ **连接失败**: 检查 API 密钥和网络连接

## 功能详解

### Claude 供应商管理

Claude 供应商管理用于配置 Anthropic Claude API 服务。

**配置字段说明**:
- **名称**: 供应商的显示名称
- **API 地址**: Claude API 的服务器地址
- **API 密钥**: 从 Anthropic 官网获取的 API 密钥
- **最大令牌数**: 单次请求的最大 token 数量
- **温度**: 模型生成的随机性参数 (0.0-1.0)
- **模型**: 使用的 Claude 模型版本
- **超时时间**: 请求超时时间（秒）
- **重试次数**: 失败重试次数

**操作说明**:
- **添加**: 点击"添加供应商"按钮创建新配置
- **编辑**: 点击供应商卡片上的"编辑"按钮
- **删除**: 点击供应商卡片上的"删除"按钮
- **启用/禁用**: 点击供应商卡片上的开关按钮
- **测试**: 点击供应商卡片上的"连接测试"按钮

### Codex 供应商管理

Codex 供应商管理用于配置各种代码生成和编程辅助服务。

**支持的服务类型**:
- GitHub Copilot
- OpenAI Codex
- 其他兼容的代码生成服务

**配置方法与 Claude 供应商类似**，请参考上一节说明。

### Agent 指导文件管理

Agent 指导文件用于管理 AI 助手的系统提示词和指导模板。

**创建指导文件**:
1. 点击"添加指导文件"按钮
2. 填写以下信息：
   - **名称**: 指导文件的名称
   - **类型**: 选择指导文件类型（如"代码助手"、"创意写作"等）
   - **内容**: 详细的指导内容和提示词
3. 点击"保存"

**最佳实践**:
- 使用清晰、具体的指令
- 包含期望的输出格式说明
- 定期更新和优化指导内容

### MCP 服务器管理

MCP (Model Control Protocol) 服务器管理用于配置和管理外部模型服务连接。

**配置步骤**:
1. 输入服务器名称和描述
2. 配置服务器连接信息
3. 设置认证方式（如需要）
4. 测试连接并保存配置

### 通用配置

通用配置管理应用级别的全局设置。

**常用配置项**:
- **默认语言**: 设置界面语言
- **主题**: 选择应用主题（浅色/深色）
- **日志级别**: 设置日志详细程度
- **数据备份**: 配置自动备份设置
- **网络代理**: 配置代理服务器设置

## 配置管理

### 数据存储位置

应用配置和数据存储在以下位置：

**Windows**: `%APPDATA%/ai-manager/`
**macOS**: `~/Library/Application Support/ai-manager/`
**Linux**: `~/.config/ai-manager/`

### 数据备份

**自动备份**:
- 应用默认每天创建一次数据备份
- 保留最近 7 天的备份文件
- 备份文件位于数据目录下的 `backups/` 文件夹

**手动备份**:
1. 在"通用配置"中找到"数据备份"选项
2. 点击"立即备份"按钮
3. 选择备份文件保存位置

### 数据恢复

**从备份恢复**:
1. 将备份文件复制到数据目录
2. 重启应用，应用会自动检测并提示恢复
3. 确认恢复操作

### 配置文件格式

主要配置文件包括：
- `settings.json`: 应用主配置
- `providers.json`: 供应商配置
- `agents.json`: 指导文件配置
- `mcp_servers.json`: MCP服务器配置
- `ai_manager.db`: SQLite 数据库文件

## 故障排除

### 常见问题

**Q: 应用启动失败**
- 检查系统是否满足最低要求
- 确认没有其他应用占用端口
- 尝试以管理员权限运行

**Q: API 连接失败**
- 验证 API 密钥是否正确
- 检查网络连接状态
- 确认 API 服务是否正常运行

**Q: 配置文件丢失**
- 检查数据目录是否存在
- 尝试从备份恢复
- 重新初始化配置

**Q: 应用运行缓慢**
- 检查系统资源使用情况
- 关闭不必要的后台应用
- 重启应用清理缓存

### 错误代码说明

| 错误代码 | 说明 | 解决方案 |
|---------|------|----------|
| E001 | 数据库连接失败 | 检查数据库文件权限 |
| E002 | API 认证失败 | 验证 API 密钥 |
| E003 | 网络连接超时 | 检查网络设置 |
| E004 | 配置文件损坏 | 从备份恢复或重置 |

### 日志文件

应用日志文件位置：
- Windows: `%APPDATA%/ai-manager/logs/`
- macOS: `~/Library/Logs/ai-manager/`
- Linux: `~/.local/share/ai-manager/logs/`

日志文件命名格式: `ai-manager-YYYY-MM-DD.log`

### 性能优化

**提升应用性能**:
1. 定期清理缓存数据
2. 优化供应商配置（如调整 token 限制）
3. 关闭不必要的实时同步功能
4. 使用 SSD 存储数据

**减少内存占用**:
1. 关闭不用的供应商配置
2. 定期重启应用
3. 减少同时打开的指导文件数量
